Pillar 3 — Evolve · R2

Restocking, co-designed with operations

Anchored in the standard inventory framework — (s, S) policy with newsvendor service-level calibration (Silver, Pyke & Thomas, 2017) — and shaped by VP Operations and her buyers in three stages.

W4 — Discovery

Listen first

Working sessions with Tanaka and 2–3 buyers. Capture the heuristics they already use.

W5–W6 — Build

Iterate weekly

Weekly demo, course correct, ship behind a feature flag.

W7 — Validate

Compare to judgement

Recommendations on a sample of SKUs; operations flags agreement, disagreement, or insight.

Desired items — co-designed, optional

D1 · D2 · D3 sized as standalone increments

  • D1 — UI modernization. Refresh on the existing token system, co-designed with operations. Not a rewrite to an external design system.
  • D2 — Internationalization. Complete the Japanese locale across all views; the i18n infrastructure is already in place.
  • D3 — Dark mode. Operator-selectable theme via existing CSS custom properties — designed for low-light warehouse stations.

Each priced separately, each can be selected, deferred, or declined independently.
